# Документ: выполнение задания по списку задач и описание файлов в папке `app`

## 1. Цель работы

Реализовать на главном экране приложения (Expo Router, React Native) функциональность по заданию:

- список задач с добавлением через `TextInput`;
- отображение всех задач;
- у каждой задачи — рабочая кнопка удаления;
- при пустом вводе и нажатии «Додати» — сообщение об ошибке через `Alert`;
- после успешного добавления — уведомление через Toast (на Android) или эквивалент на других платформах.

## 2. Как выполнено задание

**Модель данных.** Введён тип `Task` с полями `id` (уникальный строковый идентификатор) и `title` (текст задачи). Идентификатор формируется из временной метки и случайного фрагмента, чтобы при быстром добавлении нескольких задач ключи в списке не совпадали.

**Состояние интерфейса.** Используются два состояния React: массив `tasks` для списка и строка `newTaskText` для текста в поле ввода новой задачи.

**Добавление задачи (`addTask`).** Перед добавлением вызывается `trim()` у строки ввода. Если после обрезки пробелов строка пустая, показывается `Alert.alert` с заголовком «Помилка» и поясняющим текстом (требование задания). Если текст непустой, в массив задач добавляется новый элемент, поле ввода очищается, затем вызывается функция успешного уведомления.

**Уведомление об успехе (`showTaskAddedToast`).** На платформе Android используется `ToastAndroid.show` с короткой длительностью и текстом «Задачу додано успішно». В React Native для iOS и веб нативного Toast нет, поэтому для единообразия с уже существующим в проекте приёмом «демо-тоаста» используется краткий `Alert.alert` с тем же сообщением (пустой заголовок, текст в теле диалога).

**Удаление задачи (`removeTask`).** По `id` задачи массив обновляется через `filter`, элемент с соответствующим идентификатором исключается из состояния.

**Интерфейс.** Блок со списком задач оформлен отдельной карточкой: заголовок «Список задач», поле ввода с плейсхолдером «Нова задача», кнопка «Додати», ниже — список строк; у каждой строки текст задачи и кнопка «Видалити». Если задач нет, выводится текст «Поки немає задач». Весь экран обёрнут в `ScrollView` с `keyboardShouldPersistTaps="handled"`, чтобы при открытой клавиатуре можно было нажимать кнопки без лишних сбоев фокуса.

**Сохранённый учебный контент.** Ранее существовавшие на экране элементы (приветствие, подписка, демо Toast, переключатель уведомлений) не удалялись, чтобы не расширять объём работы за пределы формулировки задания; новый функционал добавлен отдельным блоком.

## 3. Описание файлов в папке `app`

В проекте используется файловая маршрутизация **Expo Router**: структура папок и имён файлов в `app` задаёт маршруты приложения.

### 3.1. `app/_layout.tsx`

Корневой layout всего приложения. Подключаются полифиллы/инициализация `react-native-reanimated` и `react-native-gesture-handler`. Корневой UI оборачивается в `GestureHandlerRootView` для корректной работы жестов. Внутри объявляется навигационный **Stack** из `expo-router`: единственный явно настроенный экран — группа `(tabs)` с отключённым шапочным заголовком (`headerShown: false`), чтобы заголовки настраивались на уровне вкладок.

### 3.2. `app/(tabs)/_layout.tsx`

Layout группы маршрутов с нижними вкладками (**Tabs**). Задаётся активный цвет вкладок и стиль шапки. Описаны две вкладки: `index` (заголовок «Main Page», иконка дома из `Ionicons`) и `about` (заголовок «About», иконка вопроса из `EvilIcons`). Имя папки `(tabs)` в скобках означает **группу маршрутов** без префикса в URL; она объединяет табы в одну навигационную структуру.

### 3.3. `app/(tabs)/index.tsx`

Экран главной вкладки (маршрут по умолчанию для группы табов). Содержит демонстрационные элементы интерфейса (карточка приветствия, кнопки с `Alert`/`Toast`, стандартная кнопка `Button`, переключатель `Switch`) и **реализованный по заданию блок списка задач**: ввод текста, добавление с валидацией и Toast/Alert, отображение списка, удаление по кнопке. Экспортируется компонент по умолчанию как экран маршрута `index`.

### 3.4. `app/(tabs)/about.tsx`

Экран второй вкладки «About». Отображает `FlatList` над пустым массивом данных `data` (тип элементов `Item` с `id` и `title`); при отсутствии элементов показывается `ListEmptyComponent` с текстом «List is empty». Закомментирован пример заполнения списка тестовыми элементами. Стили оформления согласованы с карточным видом (контейнер, карточка, элементы списка). Предназначен как отдельная страница «О приложении» или площадка для списков.
